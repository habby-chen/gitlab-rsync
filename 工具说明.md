## 一、工具功能说明

- 仅对源仓库只做读取、拉取操作。
- 支持目标仓库项目自动创建。
- 支持目标仓库分支自动创建与同步删除。
- 支持目标仓库标签自动创建与同步删除。
- 支持目标仓库与源仓库文件同步变动。
- 支持本地缓存，避免每次重启后全量从头拉取。
- 支持配置文件动态修改，无需重启服务。
- 支持一库对多库自由配置。
- 支持黑白名单配置，对同步目标过滤。

## 二、基本环境安装

#### 1、基础环境

- centos7.6及以上、2核8G，存储不小于源库代码存储容量。
- 建议 git 1.8.3
- 建议python3.7及以上

#### 2、pyhton3离线安装

- cd /data
- wget http://cdn.npm.taobao.org/dist/python/3.7.8/Python-3.7.8.tgz
- tar -zxvf Python-3.7.8.tgz 
- mv Python-3.7.8 python3
- cd python3 
- ./configure --prefix=/usr/local/python3 --with-ssl
- make && make install
- ln -s /usr/local/python3/bin/python3 /usr/bin/python3
- ln -s /usr/local/python3/bin/pip3 /usr/bin/pip3

#### 3、依赖库安装

pip3 install gevent flask apscheduler  -i http://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com

## 三、工具结构说明

#### 1、目录树结构

gitsync
├── com
│   ├── gitevents.py
│   ├── gittools.py
│   ├── loadconf.py
│   └── logger.py
├── conf.py
├── gitsync.py
└── tasks
    └── 1.json

#### 2、文件及参数说明

- **gitsync/conf.py**

  工具自身运行状态相关配置，示例如下:

TASK_DIR='/opt/gitlab/gitsync/tasks'
WORK_DIR='/opt/repo'
LOG_DIR='/opt/gitlab/gitsync/logs'
LOAD_CONF_TIMER=10
WEBHOOK_PORT=6000
TASK_RUN=[]
FILE_MD5=[]
CONFIG_FORMAT={
"source_url":str,"source_token":str,"target_url":str,"target_token":str,"target_space":str,"event_url":list,
"privilege_list":list,"privilege_list_type":str,"source_ssh":str,"target_ssh":str,"force_push_all":bool,"force_push_list":list
}
SECRET='JD98Dskw=23njQndW9D'

| 参数名             | 参数类型 | 参数说明                                                |
|-----------------|------|-----------------------------------------------------|
| TASK_DIR        | 字符串  | 工具运行的任务配置文件目录，其下文件名必须以".json"结尾，内容满足CONFIG_FORMAT格式 |
| WORK_DIR        | 字符串  | 本地仓库位置                                              |
| LOG_DIR         | 字符串  | 日志存储目录，日志每天0点分割                                     |
| LOAD_CONF_TIMER | 整数   | 动态加载配置文件时间间隔，单位S                                    |
| WEBHOOK_PORT    | 整数   | webhook配置监听端口                                       |
| TASK_RUN        | 列表   | 用于程序存放锁定的任务，保证同一个本地目录同时只能被一个任务操作，程序自身管理，无需配置        |
| FILE_MD5        | 列表   | 用于程序存放当前所加载的配置文件标识，程序自身管理，无需配置                      |
| SECRET          | 字符串  | gitlab上webhook的secret值，不配置所发送事件无法解析                 |
| CONFIG_FORMAT   | 字典   | TASK_DIR下配置文件中内容的格式标准，用于格式校验                        |

- **gitsync/gitsync.py**

  工具入口，启动后先根据配置文件全量同步，完成后再进行事件监听和配置文件监听。

- **gitsync/com/gitevents.py**

  接收的各类事件分析后结合配置文件进行预处理。

- **gitsync/com/gittools.py**

  对gitlab组、项目、分支、标签等处理调用。

- **gitsync/com/loadconf.py**

  加载配置文件。

- **gitsync/com/logger.py**

  日志Handler

- **gitsync/tasks/2.json**

  gitlab相关信息配置文件，为工具提供任务配置，位于tasks下，文件名可任意取但必须以".json"结尾，内容字段需与gitsync/conf.py中CONFIG_FORMAT匹配，支持文件的动态增删和内容修改，示例如下:

  {
  "source_url":"http://192.168.10.32:8929","source_token":"xSy1NYNTzXSEEydvyFRx","source_ssh":"","event_url":[],
  "target_url":"https://192.168.10.36:8888","target_token":"glpat-CCmJHTjsYJUpX5LYiYnC","target_ssh":"ssh://git@192.168.10.36:8889",
  "target_space":"","privilege_list":["mygroup"],"privilege_list_type":"white",
  "force_push_all":true,"force_push_list":[]
  }
  
  | 参数名                 | 参数类型 | 参数说明                                                                                                                               |
  |---------------------|------|------------------------------------------------------------------------------------------------------------------------------------|
  | source_url          | 字符串  | 源gitlab库http地址，http(https)://$IP:$PORT形式，用于拉取。                                                                                     |
  | source_token        | 字符串  | 源gitlab库用户token，用于访问源gitlab api或进行http方式拉取。                                                                                        |
  | source_ssh          | 字符串  | 源gitlab库ssh地址，ssh://git@$IP:$PORT形式，不为空且以ssh://git@开头则自行采用ssh方式拉取。                                                                 |
  | event_url           | 列表   | 事件所传url中的域名或IP:PORT，用于判断接收事件来源于哪个源gitlab。                                                                                          |
  | target_url          | 字符串  | 目标gitlab库http地址，http(https)://$IP:$PORT形式，用于推送。                                                                                    |
  | target_token        | 字符串  | 目标gitlab库token，用于访问目标gitlab api或进行http方式推送。                                                                                        |
  | target_space        | 字符串  | 拉取后推送到目标gitlab库的指定空间下，若不指定，目标库的空间将与原库一致。                                                                                           |
  | target_ssh          | 字符串  | 目标gitlab库ssh地址，ssh://git@$IP:$PORT形式，不为空且以ssh://git@开头则自行采用ssh方式推送。                                                                |
  | privilege_list_type | 字符串  | 指定黑白名单，值"white"表示白名单，非白名单即为黑名单。                                                                                                    |
  | privilege_list      | 列表   | 名单列表，可以为space、space/project、space/project/branch，优先级由高到低依次为名称空间下、项目下、分支下有效。                                                        |
  | force_push_all      | 布尔   | 是否所有强制推送，若是则无论force_push_list如何配值都将全部强制推送，在push失败后会先将目标库对应项目分支删除再次push，注意默认分支无法删除。                                                 |
  | force_push_list     | 列表   | 强制推送列表，在push失败后会先将目标库对应项目分支删除再次push，可以为space、space/project、space/project/branch，优先级由高到低依次为名称空间下、项目下、分支下有效，注意目标库的default(默认)分支无法删除。 |

## 四、启动

#### **1、启动文件**

vi /etc/systemd/system/gitsync.service 

[Unit]
Description=gitsync deamon
After=rc-local.service

[Service]
Type=simple
User=root
Group=root
ExecStart=/usr/bin/python3 /data/gitsync/gitsync.py
Restart=always

[Install]
WantedBy=multi-user.target

#### 2、启动

systemctl start gitsync



## 五、后续优化

1、源仓库删除项目或组时时，目标仓库对应项目或组不会同步删除。

原因:gitlab同步过来的project_destroy或group_destroy事件未包含源gitlab的url信息，多个源gitlab时暂无法确定是哪删除的项目，若有需求再想办法解决。

2、增加程序对必须参数的校验和问题日志反应，比如当接收的事件中的项目是源token无权限访问时的判断日志输出。
